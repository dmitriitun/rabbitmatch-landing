# Solution section

«Our Solution / Наше решение» — секция-объяснение продукта между Hero и остальной частью лендинга. Закреплена за якорем `#features` (на него ведёт пункт меню `Features`).

## Что внутри

Двухколоночная композиция:

| Левая колонка | Правая колонка |
|---|---|
| Eyebrow-заголовок «Our Solution», лидер-параграф, заголовок «The RabbitMatch Platform», три нумерованные карточки (`01`, `02`, `03`) с заголовками и описаниями. | Фотореалистичный mockup телефона + стилизованный CRM-мокап + фиолетовый glow-бейдж «AI that helps you earn money». |

Контент берётся из `messages/{en,ru}.json` под ключом `solution.*`.

## Архитектура компонентов

```
src/components/
  Solution/
    Solution.tsx          (server) — собирает текст и мокапы
    Solution.module.css
    CrmMockup.tsx         (server) — HTML/CSS дашборд
    CrmMockup.module.css
  ScrollReveal/
    ScrollReveal.tsx      (client) — переиспользуемый fade-in
    ScrollReveal.module.css
```

### Почему CRM — HTML, а не PNG

Картинки CRM-панели нужного качества под рукой нет, а тяжёлый PNG плох по двум причинам: вес (2–3 MB) и фиксированный pixel-aspect (выглядит мыльно на ретине, плохо адаптируется). `CrmMockup.tsx` рендерит верхнюю часть CRM (сайдбар с навигацией, KPI-карточки, расписание кортов с busy/free-полосами, ленту последних бронирований) на чистом HTML+CSS. Это:

- адаптивно (на узких экранах сжимаются колонки таблицы),
- меньше 5 KB разметки + стилей,
- iconography из `lucide-react` совпадает с остальным сайтом.

Мобильный mockup — реальный PNG `/images/phone-courts.png` (взят из `~/Desktop/RM/presentation and screen/apple-iphone-15-black-mockup/...`), потому что хочется именно физическую рамку iPhone с тенью.

## ScrollReveal

Переиспользуемая клиентская обёртка. Принимает `delayMs`, `offset`, `as`. Логика:

1. Маунтит `IntersectionObserver` (`threshold: 0.12`, `rootMargin: '0px 0px -60px 0px'` — срабатывает чуть до полного входа в видимую область).
2. На первом пересечении вызывает `setShown(true)` (в колбэке обсервера — не в теле эффекта, что соответствует React 19.2 правилу `react-hooks/set-state-in-effect`).
3. Стартовое состояние: `opacity: 0; transform: translateY(var(--reveal-offset))`. На `.shown` — финальное.
4. При `prefers-reduced-motion: reduce` CSS форсит `opacity: 1; transform: none; transition: none`, поэтому JS-логика остаётся без изменений: мгновенный fade-in без анимации.

В Solution используется с возрастающими `delayMs` (0 / 80 / 140 / 220 / 320 / 420 / 120 / 260 / 420) — даёт каскадное появление сверху вниз.

## Визуальные эффекты

- **Фон секции:** два больших радиальных glow — лайм (12% / 10%) и фиолетовый (90% / 80%) — задаются через `radial-gradient` на `.bgGlow`, добавляют объёма.
- **Glow вокруг телефона:** мягкий лайм-radial с `filter: blur(40px)` за самой картинкой.
- **CRM glow:** два цветных blur-пятна (лайм в верхнем правом, фиолетовый в нижнем левом углах рамки) — повторяют референс-слайд.
- **AI-бейдж:** фиолетовый pill (`#4300BD` основной, светло-фиолетовый бордер `rgba(167, 139, 250, 0.35)`). Три слоя свечения через `box-shadow` (внешнее 40px + 80px + внутреннее 24px). Пульсирует `aiPulse 3.6s ease-in-out infinite` — `box-shadow` плавно увеличивается и возвращается. Отключается при `prefers-reduced-motion`.
- **Карточки 01/02/03:** при hover лайм-бордер, тёмный фон чуть ярче, лёгкий подъём `translateY(-1px)`.

## Адаптив

- **<960px:** одна колонка. Сначала текст и нумерованные блоки, ниже — телефон по центру, ещё ниже — CRM, в конце — AI-бейдж по центру.
- **≥960px:** две колонки. Телефон сдвинут влево (`margin-left: -20px`), CRM сдвинут вправо вверх (`margin-left: 24%; margin-top: -120px`) — повторяет диагональную композицию с референс-слайда. AI-бейдж прижат к правому нижнему углу правой колонки.
- **<480px:** нумерованная колонка карточки сужается до 32px.

## Интеграция

`src/app/page.tsx` рендерит `<Solution />` сразу после `<Hero />`. Из массива плейсхолдеров убран `features`, потому что Solution сам владеет якорем `#features`.
