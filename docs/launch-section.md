# Launch in 3 Steps section

Секция онбординга клуба. Закреплена за якорем `#pricing` (пункт меню «Pricing») — это самая близкая по смыслу точка входа: как именно клуб подключается к платформе.

## Контент

- **Eyebrow:** `Onboarding / Подключение`
- **Заголовок:** `LAUNCH IN 3 STEPS / Запуск за 3 шага` (uppercase, разреженное `letter-spacing: 0.04em`, градиент белый→светло-серый).
- **Два intro-параграфа** с лаймовыми зачинами (стиль референса):
  - «Full localization in your language,…» — про английский/русский, поддержку 24/7.
  - «A quick start in 48 hours.» — про настройку под клуб и персонального менеджера.
- **Три шага** в списке `<ol>` с лайм-номерами `01`/`02`/`03` (`text-shadow: 0 0 24px rgba(185,233,1,0.35)` — мягкий glow):
  - **01.** Настройка системы — брендинг, корты, тарифы, расписание.
  - **02.** Импорт базы клиентов — контакты, история, абонементы.
  - **03.** Обучение команды — онлайн-обучение, поддержка 24/7.

Справа — SVG-арт бегуна по ступеням.

## Дерево компонентов

```
src/components/
  Launch/
    Launch.tsx           (server) — собирает текст и шаги
    Launch.module.css
    RunnerArt.tsx        (server) — inline-SVG бегун с лайм-обводкой
```

## RunnerArt

Inline-SVG `320×420`. Все линии нарисованы как `polyline` и `circle` с прозрачной заливкой, обводка `currentColor` = `var(--color-lime)`, `stroke-width: 8`, `stroke-linecap: square`, `stroke-linejoin: miter` — угловатые сочленения «как на референсе».

Состав:

- голова — `<circle>`
- ступени под ногами — длинный `<polyline>` (низ → правая верхняя ступень)
- спина-корпус — `<polyline>` от шеи через торс к тазу
- ведущая нога (поднята на следующую ступень)
- опорная нога (стоит на нижней ступени)
- две руки — одна назад, вторая вперёд

Поверх группы навешан SVG-фильтр `<feGaussianBlur stdDeviation="6"> + feMerge` — даёт «неоновое» свечение прямо в обводке. Сверху ещё CSS `drop-shadow(0 0 18px rgba(185,233,1,0.5))` и radial-blur за фигурой (`.runnerGlow`).

Анимация `runnerFloat` — фигура плавно качается ±12px по вертикали (6s ease-in-out). Отключается при `prefers-reduced-motion: reduce`.

## Сетка

- **<960px:** одна колонка, бегун ниже текста, центрирован.
- **≥960px:** две колонки `1.05fr / 1fr`, бегун прижат к правому краю — повторяет композицию референс-слайда.
- **<480px:** колонка с номером шага сжимается до 44px, padding карточек уменьшается.

## Анимация появления

Через `ScrollReveal` — переиспользуем компонент из секций Solution/Players. Каскадные `delayMs`: eyebrow → title → intro1 → intro2 → шаги (460/600/740ms) → бегун (220ms, параллельно с title). На reduced-motion CSS форсит мгновенный показ.

## Интеграция

- В [src/app/page.tsx](../src/app/page.tsx) добавлен `<Launch />` после `<Players />`.
- Якорь `#pricing` теперь принадлежит этой секции — из массива плейсхолдеров `pricing` убран. Пункт меню «Pricing» в Header ведёт сюда.
- В `page.tsx` упразднено отдельное «исключение» для `pricing`, потому что в `PLACEHOLDER_SECTIONS` его больше нет — все ключи теперь покрываются `SECTION_LABEL_KEY` напрямую.
